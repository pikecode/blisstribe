### 2.2 技术栈（MVP）

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端 | UniApp + Vue 3 + TypeScript | 跨端开发 |
| 网关 | Nginx | 反向代理、SSL、静态资源 |
| 后端 | Node.js (Express/NestJS) | 单体业务服务 |
| 数据库 | PostgreSQL 15 | 关系型数据 |
| 缓存 | Redis 7.x | 会话、缓存、限流 |
| 文件存储 | 腾讯云 COS / 阿里云 OSS | 图片、文件 |

> **演进说明**：当前为 MVP 单体架构。后续随业务增长可拆分为微服务，引入消息队列、独立监控等。
>
> | 演进阶段 | 触发条件 | 变更 |
> |---------|---------|------|
> | P2 | DAU > 10万 | 引入 MQ 处理异步任务 |
> | P2 | 多团队并行 | 拆认证/用户/文件服务 |

## 4. 数据流

### 4.1 注册流程数据流

```
用户 ──► 小程序前端 ──► API网关 ──► 认证服务 ──► 微信服务器
                                              ──► 获取 openid/unionid
                                              ──► 返回用户信息

用户 ──► 小程序前端 ──► API网关 ──► 认证服务 ──► 微信服务器
                                              ──► 获取手机号
                                              ──► 返回 phone

用户 ──► 小程序前端 ──► API网关 ──► 后端单体服务 ──► PostgreSQL
                                              ──► 创建用户记录
                                              ──► 返回 userInfo + Token
```

### 4.2 登录态校验数据流

```
用户操作 ──► 小程序前端 ──► API网关
                              │
                              ▼
                         Token 校验 ──► Redis 查 session
                              │
                              ▼
                         校验通过 ──► 转发请求到业务服务
                         校验失败 ──► 返回 401，前端跳转登录
```

## 6. 接口规范

### 6.1 统一响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": 1704067200000,
  "requestId": "req_xxx"
}
```

### 6.2 错误响应格式

```json
{
  "code": 400001,
  "message": "微信授权失败",
  "data": null,
  "timestamp": 1704067200000,
  "requestId": "req_xxx"
}
```

### 6.3 HTTP 状态码

| 状态码 | 含义 | 场景 |
|--------|------|------|
| 200 | 成功 | 正常响应 |
| 400 | 请求参数错误 | 参数校验失败 |
| 401 | 未授权 | Token 无效或过期 |
| 403 | 禁止访问 | 权限不足 |
| 404 | 资源不存在 | URL 错误 |
| 429 | 请求过于频繁 | 限流触发 |
| 500 | 服务器内部错误 | 系统异常 |
